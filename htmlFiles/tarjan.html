What is Tarjan's Algorithm?

Tarjan's finds all Strongly Connected Components (SCCs) in a directed graph using a single DFS pass with O(V+E) complexity — more efficient than Kosaraju's two-pass approach.

How it Works

  • Assign each node a discovery time disc[].
  • Track lowest reachable disc via low[].
  • Maintain a stack of nodes currently in DFS path.
  • When low[u] == disc[u] → u is the root of an SCC.
  • Pop all nodes from stack until u → they form one SCC.

Color Legend

  • 🟡 Yellow — DFS active (being explored)
  • 🟢 Green — visited, on DFS stack
  • 🔴 Red edge — back edge (part of cycle)
  • 🎨 Final color per node = SCC group

Tarjan's vs Kosaraju's

  • Tarjan's: single DFS pass, one graph.
  • Kosaraju's: two DFS passes, needs reversed graph.
  • Both are O(V + E).

Time & Space Complexity

  • Time: O(V + E)
  • Space: O(V)
