What is Bellman-Ford?

Bellman-Ford computes shortest paths from a source to all vertices. Unlike Dijkstra, it handles negative edge weights and can detect negative cycles.

How it Works

  • Initialize dist[src]=0, all others=∞.
  • Repeat V-1 times: relax every edge (u,v,w).
  • Relaxation: if dist[u]+w < dist[v], update dist[v].
  • V-th pass: if any update occurs → negative cycle!

Color Legend

  • Yellow — source node
  • Orange — distance just updated/relaxed
  • Red — negative cycle detected
  • Label below = current best distance

Dijkstra vs Bellman-Ford

  • Dijkstra: O((V+E)logV), no negative weights.
  • Bellman-Ford: O(VE), supports negatives + cycle detection.

Time & Space Complexity

  • Time: O(V × E)
  • Space: O(V)

Applications

  • Routing protocols (RIP — distance vector).
  • Currency arbitrage detection.
  • Shortest paths with negative weights.
