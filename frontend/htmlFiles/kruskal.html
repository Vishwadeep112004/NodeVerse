What is Kruskal's Algorithm?

Kruskal's builds a Minimum Spanning Tree by greedily picking the smallest weight edge that does not form a cycle, using a Union-Find (DSU) data structure to track connected components.

How it Works

  • Sort all edges by weight (ascending).
  • For each edge (u, v, w):
  • → If find(u) ≠ find(v): add to MST, unite(u, v).
  • → Else: skip (would create a cycle).
  • Stop when MST has V-1 edges.

Color Legend

  • Active edge (yellow) — edge being considered
  • 🔴 Red edge — rejected (would create cycle)
  • 🟢 Green node — added to MST
  • Active + green = final MST edge accepted

Union-Find (DSU)

  • Uses path compression + union by rank.
  • Nearly O(1) per operation (inverse Ackermann).
  • Efficiently checks if two nodes are already connected.

Time & Space Complexity

  • Time: O(E log E) — dominated by sorting
  • Space: O(V) — DSU arrays

Applications

  • Minimum cost network design.
  • Image segmentation in computer vision.
  • Approximation for TSP.
  • Clustering algorithms.
